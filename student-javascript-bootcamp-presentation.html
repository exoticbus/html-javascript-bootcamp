Key ideas to remember

JavaScript goes in a script tag or external .js file

External files are linked with src, usually near the end of the body.

JavaScript can change the page

It can find elements, read input values, and update text when an event happens.

Task 1

Debug the broken code

🔎

Spot errors

Look for wrong file names, IDs, and missing punctuation.

🧩

Fix logic

Make sure the function runs and updates the output correctly.

✅

Test it

Type a name, click the button, and check the result message.

Broken HTML sample

<input type="text" id="username" placeholder="Enter your name"> <button id="greetBtn">Show Greeting</button> <p id="outputText"></p> <script src="app.jss"></script>

Can you spot the file problem before running the code?

Broken JavaScript sample

let button = document.getElementById("greatBtn"); let nameBox = document.getElementById("userName"); function showGreeting { let name = nameBox.value if (name == "") { output.innerHTML = "Please enter your name" else { output.innerHTML = "Hello, " + name + "!" } } button.addEventListner("click", showGreeting);

Look for ID mistakes, missing brackets, and a misspelled method name.

What you need to fix

HTML

  • Make sure the JavaScript file name is correct.
  • Check that the HTML IDs match the JavaScript selectors.
  • Keep the output paragraph ready for the result.

JavaScript

  • Fix function syntax and missing punctuation.
  • Use the correct element IDs.
  • Use addEventListener correctly.
  • Show the message in the output area.
